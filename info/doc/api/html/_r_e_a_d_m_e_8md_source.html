24 # Purpose
25 
26 The BioCASe Monitor Service (BMS) is a tool for coordinators of networks of biodiversity databases
27 that are based on the BioCASe Provider Software (BPS).
28 
29 The backend part of the tool
30 allows registration of BioCASe providers and their data sources in a user-friendly UI.
31 
32 In the frontend, all relevant information are displayed in an aggregated view, e.g.
33 - provider name (linking to the access point URL),
34 - list of data sources, total number of records and of unique values per concept (i.e. number of units, number of multimedia objects),
35 - link to the consistency checker tool
36 - link to archives
37 - other useful links.
38 
39 In the EU funded project OpenUp! the BioCASe Monitor Service is used to monitor the progress of the provision of multimedia objects to Europeana.
40 
41 Furthermore, it is a useful tool for the coordinators of the content-providing work packages
42 as a quality check of the mapping of the associated metadata in the ABCD or ABCDEFG schema
43 and its required compliance with the Europeana Sematic Elements (ESE), the Europeana standard.
44 The tool has been developed in close collaboration between the OpenUp! team (Work Package 4 & 7)
45 and the GBIF-D team at the Museum f√ºr Naturkunde Berlin, Germany.
46 
47 The version series 2.x code has been completely rewritten, and relies on a sqlite3 database.
48 
49 A more detailed description has been published in the [Biodiversity Data Journal] (https://www.ncbi.nlm.nih.gov/pmc/articles/PMC3964725/)

57 # Installation
58 
59 
60 ## Requirements
61 - Webserver, e.g. apache2
62 - Database engine, e.g. sqlite3
63 - PDO
64 - PHP 5.6
65 - php5-curl, php5-sqlite, php5-xsl
66 
67 ## Installation steps
68 1. copy everything to a webfolder
69 2. edit configuration file `config/config.php`
70 3. set files permissions
71  - all files must be readable by webuser
72  - DB file `provider.sqlite` must be writable by webuser
73  - Cache folder `data_cache` must be writable by webuser
74 4. create and populate database
75  - run statements defined in `install/sampledata.sql`
76 
77 The following shell script `install/install.sh` does the job for an installation on linux.
78 Maybe you are required to have sudoer's rights:
79 
80  #!/bin/bash
81  # please edit these settings
82  DBDIR=/my/db/folder/
83  WEBDIR=/var/www/bms
84  WEBUSER=www-data
85  # running the commands
86  mkdir $DBDIR
87  mkdir $WEBDIR
88  cd $WEBDIR
89  chgrp -R $WEBUSER .
90  chmod -R g+w $DBDIR
91  mkdir data_cache
92  chmod g+w data_cache
93  cd config
94  ln -s config.sample.php config.php
95  mv sampledata.sql $DBDIR
96  sqlite3 $DBDIR/provider.sqlite < $DBDIR/sampledata.sql
97  chgrp $WEBUSER $DBDIR/provider.sqlite

108 ## Typical Administrator's Workflow
109 
110 1. Login
111 
112  * Login via the navigation bar
113  * type in your credentials and validate
114  * select a provider in the dropdown list
115 
116 2. Main Metadata
117 
118  * edit the provider's Main Metadata
119 
120 3. Count Concepts
121 
122  * example: `/DataSet/Units/Unit/MultiMediaObjects/MultiMediaObject/FileURI`
123  * modify a concept: autocompletion from all concepts defined in ABCD2
124  * move a concept up and down, using the handle
125  * save the current concept count
126  * delete the current concept count
127  * add a new concept count
128 
129 4. Data Source Access Points (DSAs)
130 
131  - Data Sources
132  * Data Sources are presented in tabs: a tab for each data set, one data source visible
133  * operations: `add`, `delete`, `save`
134  * edit required fields
135  * status: active/inactive
136  * data-source: select select a specific Data Source in the dropdown list
137  * title: type a human readable full title
138  * data-set: select a specific Data Set in the dropdown list
139  * landing-page: select type of landingpge:
140  * automatically generated: this is the default landingpage, based on a scan request for the concept `/DataSets/DataSet/Units/Unit/MultiMediaObjects/MultiMediaObject/FileURI`.
141  * user-defined URL: enter an URL
142  * for advanced users only:
143  * filter: define a complex filter
144  * Beware ! The filter is applied **as is** (not validated) to the request sent to the pywrapper
145 
146  - Archives
147  * add an arbitrary list of archives
148  * an archive is a URL pointing to a downloadable file
149  * one archive must be marked as "latest"
150 
151  - Useful Links
152  * add an arbitrary list of links
153  * a link is a URL pointing to a downloadable file and should be one of the following types:
154  * BioCASe Query Tool
155  * GBIF
156  * Europaeana
157  * Deutsche Digitale Bibliothek
158  * GeoCASe
159  * BiNHum
160  * Pangaea
161  * GFBio
162  * example: `http://www.gbif.org/dataset/71f03224-f762-11e1-a439-00145eb45e9a` (GBIF)
163 
164 7. Logout
165 
166  * log out via the navigation bar
